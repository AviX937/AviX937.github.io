# Спека дизайна: Portfolio site refresh (A+B+C)

Дата: 2026-05-23  
Проект: статический сайт-портфолио на GitHub Pages (Jekyll, без плагинов)  
Цель: обновить визуал в сторону Swiss-типографики, добавить тёмную тему (auto + переключатель), добавить витрину дизайн-проектов, исправить баги и привести репозиторий к актуальной архитектуре (one-page + translations).

## 1) Контекст и ограничения

### Текущая реализация
- Jekyll (штатная сборка GitHub Pages), layout: `_layouts/default.html`.
- Главная страница: `index.html` (one-page: hero, toc, about, portfolio, contact).
- Переводы: `translations/*.md` (рендерятся Jekyll).
- Локализация: `js/i18n.js`, применение: `js/main.js`, сохранение языка: `localStorage`.
- Анимация появления ссылок в хедере при уходе ToC: IntersectionObserver.
- Подсветка активного пункта ToC и хедера: scroll handler.
- Дизайн: минимализм (ч/б) + розовый акцент Dribbble.

### Жёсткие ограничения
- Статика, без внешних API, без тяжёлых фреймворков, без платных сервисов.
- Максимально лёгкий JS (без библиотек).
- Должно корректно работать на GitHub Pages (в том числе при `baseurl`).
- Доступность в РФ: отказаться от Google Fonts в пользу self-host woff2.

## 2) Scope / Non-goals

### В scope (делаем)
1) Clean-up репозитория под актуальную архитектуру:
- Удалить устаревшие многостраничные файлы: `about.html`, `portfolio.html`, `services.html`, `contact.html`.
- Починить/добавить `404.html`.
- Привести `translations/*.md` к валидному виду (front matter + контент без “инструкций”).

2) Багфиксы и устойчивость:
- Исправить формирование ссылок (резюме, якоря навигации) так, чтобы работало:
  - на главной `/` и на страницах `/translations/...`;
  - при `baseurl` на GitHub Pages.
- Сделать сбор ссылок для подсветки менее хрупким (не завязываться на `href^="/#"`).
- Сделать вычисление высоты хедера динамическим, чтобы IntersectionObserver корректно работал на мобильных.

3) Визуальный refresh (Swiss/типографика):
- Перевести стиль на систему CSS-токенов (переменные) для типографики/цветов/отступов.
- Усилить визуальную иерархию: крупный hero, более строгая сетка, больше воздуха.
- Улучшить состояния наведения/фокуса (доступность).

4) Тёмная тема:
- Авто по `prefers-color-scheme`.
- Переключатель темы в UI с сохранением выбора пользователя в `localStorage`.
- Приоритет: выбор пользователя > системная тема.

5) Витрина дизайн-проектов (галерея):
- Лёгкая галерея карточек (без внешних ссылок/плейсхолдер-сервисов).
- Данные: локально (например, `data/portfolio.js`).
- Изображения: локально в `assets/` (thumb + full), либо один файл, если full не нужен.
- Просмотр: кликом открывать модал/лайтбокс с большим изображением и названием/описанием.

### Non-goals (не делаем)
- Подключение CMS/конструкторов, сторонних JS-библиотек.
- Полноценную фильтрацию/поиск по портфолио (оставляем “каркас” для будущего).
- Подмену контента-заглушек на реальные тексты (пока не готов контент).

## 3) Архитектура и файлы

## 4) Детали реализации (поведение)

### 4.1 Язык
- Хранение: `localStorage.site-lang`.
- Применение:
  - проставить `document.documentElement.lang`;
  - заменить текст во всех `[data-i18n]`;
  - обновить `href` резюме.

### 4.2 Резюме (путь)
- В разметке хранить оба пути с учётом Jekyll `relative_url`, чтобы избежать ручной сборки путей в JS.
- JS только выбирает между заранее подготовленными `href`.

### 4.3 Навигация (ToC + header)
- IntersectionObserver:
  - наблюдаем ToC;
  - при уходе ToC за верхнюю границу показываем ссылки в хедере;
  - offset брать от реальной высоты хедера.
- Подсветка активного раздела:
  - собрать ссылки из ToC и из хедера;
  - для каждой ссылки извлечь id якоря (после `#`);
  - подсветить все ссылки, чей id совпадает с текущим разделом.

### 4.4 Тема (auto + переключатель)
- Схема:
  - ключ `localStorage.site-theme` со значением `light|dark`;
  - если ключ отсутствует, берем `matchMedia('(prefers-color-scheme: dark)')`;
  - применение: `document.documentElement.dataset.theme = 'dark'|'light'`.
- UI:
  - кнопка рядом с переключателем языка;
  - текст/иконка меняется в зависимости от текущей темы.

### 4.5 Галерея дизайна
- На главной в секции “Дизайн” отображаем:
  - сетку карточек (thumb + title);
  - опционально кнопку Dribbble (как внешний профиль/архив).
- При клике по карточке:
  - если есть `href`, открываем в новой вкладке;
  - иначе открываем модал с большим изображением и текстом.
- Модал:
  - закрытие по клику на overlay и по Escape;
  - блокировка скролла body, пока открыт.

## 5) Self-host шрифта
- Добавить woff2 JetBrains Mono в `assets/fonts/`.
- Подключить в CSS через `@font-face`.
- Удалить Google Fonts ссылки из layout.
- Fallback стек: `ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, "Liberation Mono", "Courier New", monospace`.

## 6) SEO (минимальный набор)
- `meta name="description"` брать из `page.description` → fallback `site.description`.
- Добавить OpenGraph:
  - `og:title`, `og:description`, `og:type`, `og:url`.
- `title` формировать: `page.title | site.title`.

## 7) Acceptance criteria (что считается готовым)
- Сборка Jekyll проходит локально и структура корректна для GitHub Pages.
- На главной:
  - ToC виден под hero, ссылки в хедере появляются/исчезают корректно;
  - подсветка активного пункта синхронизирована ToC + header;
  - переключение языка работает и на главной, и на страницах переводов;
  - ссылка “Резюме” корректна на всех страницах независимо от baseurl.
- Тема:
  - по умолчанию соответствует системной;
  - ручной переключатель сохраняется и имеет приоритет.
- Галерея:
  - рендерится из `data/portfolio.js`;
  - карточки открывают модал/ссылку без ошибок;
  - нет внешних placeholder-изображений.
- Репозиторий очищен от устаревших страниц, `404.html` не пустой.
- `translations/*.md` рендерятся как страницы, без случайного “кода инструкции”.
